# Shared TypeScript Types

This directory contains canonical type definitions used across the application. Import from `@/types` for clean imports.

## Files

| File | Purpose | Used By |
|------|---------|---------|
| `vision.ts` | FormError, FormAnalysis, Landmark, LandmarkData, VisionWorkerMessage | Vision system, stores |
| `voice.ts` | VapiMessage, VoiceContext, VapiConnectionState, VoiceState | Voice AI integration |
| `session.ts` | SessionState, ExerciseResult, SessionMetrics, ExercisePrescription | Session tracking |
| `exercise.ts` | Exercise, ExerciseTier, BodyRegion, DifficultyLevel | Exercise catalog |
| `common.ts` | ApiResponse, ApiError, PaginatedResponse, User, UserRole | All API routes |
| `guards.ts` | Type guard functions (isFormError, isFormAnalysis, etc.) | Runtime validation |
| `character.ts` | 3D mascot character types | Character component |
| `index.ts` | Barrel export - re-exports all types | - |

## Usage

```typescript
// Preferred: Import from barrel
import type { FormError, FormAnalysis, VoiceContext } from '@/types';

// Also valid: Import from specific module
import type { FormError } from '@/types/vision';

// Type guards for runtime validation
import { isFormError, isApiError } from '@/types';
```

## Design Decisions

- **String literal unions** over enums (no runtime cost)
- **Interfaces** for objects (extensible), **types** for unions
- **JSDoc comments** on all exported types
- Store-specific state types live in their store files, not here
